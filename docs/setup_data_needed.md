## 1. Домен и хостинг

### Что нужно купить
Для реального деплоя (не localhost) нужен:

| Что | Зачем | Где купить |
|-----|-------|------------|
| **VPS-сервер** | Держит всю систему 24/7 | TimeWeb, REG.RU, DigitalOcean, Hetzner |
| **Домен** | Красивый адрес вместо IP | REG.RU, nic.ru, namecheap.com |
| **SSL-сертификат** | HTTPS (шифрование) | Бесплатно через Let's Encrypt (Certbot) |

### Минимальные требования к VPS
- CPU: 2 ядра
- RAM: 4 ГБ (минимум), рекомендую 8 ГБ
- Диск: 40 ГБ SSD
- ОС: Ubuntu 22.04 или 24.04
- Порты открыты: 80, 443

### Заполните:

```
# Домен для СИСТЕМЫ СОТРУДНИКОВ (внутренний, можно без домена — только по VPN или IP)
EMPLOYEE_HOST=sd.company.local          ← или IP-адрес сервера

# Домен для КЛИЕНТСКОГО ПОРТАЛА (публичный, клиенты будут видеть)
PORTAL_HOST=portal.company.ru           ← сюда клиентам будете давать ссылку

# IP-адрес вашего VPS (получите после покупки)
SERVER_IP=                              ← например: 185.12.34.56
```

## 2. Корпоративная почта (Outlook / Microsoft 365)

Вы упомянули корпоративный Outlook. Есть два варианта отправки через него:

### Вариант A — Outlook SMTP (проще, работает сразу)

Для корпоративного аккаунта Microsoft 365 SMTP включён по умолчанию.

```
# Заполните эти поля:
SMTP_HOST=smtp.office365.com
SMTP_PORT=587
SMTP_TLS=true

SMTP_USER=                   ← ваш корпоративный email, например: servicedesk@company.ru
SMTP_PASSWORD=               ← пароль от этого ящика

MAIL_FROM=                   ← тот же email: servicedesk@company.ru
MAIL_FROM_NAME=Service Desk  ← имя отправителя (можно изменить)
```

> **Важно:** Если в организации включена MFA (двухфакторная аутентификация),
> обычный пароль не подойдёт. Нужно создать **App Password** или попросить
> IT-администратора включить SMTP AUTH для этого ящика в Exchange Admin Center.

### Вариант B — Выделенный ящик для системы (рекомендую)

Лучше создать отдельный ящик специально для уведомлений:
- например `noreply@company.ru` или `servicedesk@company.ru`
- Попросите IT создать такой ящик
- Тогда заполняете так же как Вариант A, но с новым ящиком

### Вариант C — SendGrid (если Outlook не разрешает SMTP)

Бесплатный тариф: 100 писем/день — достаточно для старта.
1. Зарегистрируйтесь на sendgrid.com
2. Создайте API Key (Settings → API Keys)
3. Верифицируйте домен отправителя

```
SENDGRID_API_KEY=            ← вставьте ключ вида SG.xxxxxxxx
MAIL_FROM=                   ← email который верифицировали в SendGrid
MAIL_FROM_NAME=Service Desk
```

## 3. Безопасность — SECRET_KEY

Это секретный ключ для шифрования JWT-токенов. Генерируется один раз.

```
SECRET_KEY=                  ← оставьте ПУСТЫМ, я сгенерирую сам
```

Если хотите сгенерировать сами — выполните в терминале:
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

## 4. Администратор системы

Учётные данные первого администратора (создаётся автоматически при старте):

```
ADMIN_USERNAME=admin         ← можно изменить
ADMIN_PASSWORD=              ← придумайте надёжный пароль (мин. 12 символов)
ADMIN_EMAIL=                 ← email администратора (для уведомлений)
ADMIN_FULL_NAME=             ← имя администратора, например: Иванов Алексей
```

## 5. База данных (если нужна своя)

По умолчанию PostgreSQL запускается автоматически в Docker — ничего настраивать не нужно.
Если хотите подключить внешнюю БД (корпоративный PostgreSQL):

```
DB_EXTERNAL=false            ← поставьте true если хотите внешнюю БД
DB_HOST=                     ← адрес сервера БД
DB_PORT=5432
DB_NAME=servicedesk
DB_USER=servicedesk
DB_PASSWORD=                 ← пароль к БД
```
