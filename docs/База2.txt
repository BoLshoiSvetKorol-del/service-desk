## Контекст проекта

Корпоративная Service Desk система для управления заявками и инцидентами.

### Технический стек
- Backend: Python 3.12 + FastAPI
- Frontend: React 18 + TypeScript + Ant Design (antd)
- База данных: PostgreSQL
- ORM: SQLAlchemy + Alembic для миграций
- Аутентификация: JWT токены
- API: RESTful, документация через Swagger (автоматически из FastAPI)

### Функциональные требования (MVP)

#### Аутентификация и пользователи
1. Вход по логину и паролю для сотрудников
2. Роли: администратор, агент поддержки, пользователь

#### Управление заявками (тикетами)
3. Создание заявок через веб-форму, с присвоением уникального номера (формат: SD-YYYY-XXXXX)
4. Карточка заявки: данные клиента, контакты, дата/время поступления, описание, приоритет, статус
5. Типы заявок (справочник): вид услуги, направление работ
6. Приоритеты: низкий, нормальный, высокий, критичный
7. Автоматический расчёт SLA по приоритету и типу заявки

#### Назначение и обработка
8. Назначение заявки на сотрудника или отдел
9. Переназначение и эскалация
10. Статусы: новая → в работе → ожидает информации → выполнена / отменена
11. Подсветка просроченных заявок (визуальные индикаторы)

#### Хронология и файлы
12. Лента действий по заявке: изменения статусов, назначений, комментарии
13. Прикрепление файлов к заявкам и комментариям

#### Поиск и фильтрация
14. Поиск по: номеру, клиенту, дате, статусу, приоритету, исполнителю
15. Сохранение фильтров

#### Отчёты
16. Количество заявок за период
17. Распределение по статусам
18. Среднее время обработки
19. Выгрузка в Excel/CSV
